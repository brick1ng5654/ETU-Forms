# Руководосво по работе https протокола.

1) Для запуска в корне проекта должна находиться папка ssl с ключами и шифром
2) В терминале переходим а корень проекта и прописываем `docker compose up -d --build`
3) через docker ps проверяем наличие контейнера. 
4) По адресу https://localhost будет главная страница проекта, с адреса http://localhost будет редиректить на https.

## Почему может не запускаться и не работать.

У меня была проблема, что nginx был запущен в фоне в wsl, то есть надо было остановить его в wsl, `sudo systemctl stop nginx`.

Если запустился контейнер, но адреса https://localhost не работают:
1) Надо зайти внутрь контейнера (В Windows winpty docker exec -it (id контейнера из docker ps) sh)
2) Далее отправим запрос на наш адрес
    - `curl -I http://localhost` - Должно быть написано, что переадресовано и находиться на https://localhost . 301 Moved Permanently
    - `curl -k https://localhost` - Должно выдать html код страницы

Если все супер, то выходим из контейнера с помощью exit, проверяем наш хост

Повторяем все команды, должны быть абсолютно идентичные выводы, первая команда должна редиректить, а вторая возвращать html код. В дальнейшем будем работать над конфидициальностью ключей и шифров.
